Answer to the Questions

1. What is the difference between var, let, and const?

Answer: These are three ways to name a variable, but they handle data differently. var is the oldest way. It can be updated or redefined anywhere in a function, which sometimes causes bugs. let is the modern version of var. it allows the value to be changed later, but it stays strictly inside the block of code. Where it was created. Const is used for values that should never change. Once a value is assigned to a const, it cannot be renamed or pointed to something else.

2. What is the spread operator (...)?

Answer: The spread operator is written as three dots (...) and is used to spread out the contents of an array or an object. It is very helpful for making copies of data without changing the original version. For example, if there are two separate lists of numbers, the spread operator can be used to quickly combine them into one single list by spreading the items of both into a new array.

3. What is the difference between map(), filter(), and forEach()?

Answer: These are different ways to communicate to every item in a list. map() is used when a new list is needed or created. It takes the original list, changes every item in some way, and gives back a new list of the same length. filter() is used to filter a list. It looks at every item and only keeps the ones that match a certain rule, returning a shorter list of just those items. forEach() just runs a piece of code for every item in the list.

4. What is an arrow function?

Answer: An arrow function is a shorter, cleaner way to write a function in JavaScript using a specific symbol (=>). Instead of typing the word "function" every time, the arrow syntax allows the code to be written on fewer lines.

5. What are template literals?

Answer: Template literals are a way to work with text using backticks ( ` ) instead of regular quotes. They make it much easier to combine variables with text. Instead of using plus signs to join pieces of words together, a variable can be dropped directly into the text using the ${variable} format. They also allow for writing text across multiple lines without the code breaking.
